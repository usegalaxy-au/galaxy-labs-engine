Galaxy Lab Section Schema Documentation

This document describes the structure and validation rules for a Galaxy Lab sections.yml file.

Section

Description

A Galaxy Lab section - this is the root of the section.yml file.

Properties

  • id (string, required)
    Unique identifier for the section.

  • title (string, required)
    Title of the section.

  • tabs (array of SectionTab, required)
    List of tabs to be displayed for this section.

SectionTab

Description

A tab with a list of expandable items to be shown in the section.

Properties

  • id (string, required)
    Unique identifier for the tab.

  • title (string or null, optional)
    Title of the tab. Defaults to null.

  • content (array of TabItem, object, or null, optional)
    A list of content items to be displayed in this tab. This can be omitted to create a tab with arbitrary MD/HTML content using the heading_md property. Can be any of:

    • An array of TabItems.
    • An object of { 'subsections': array of TabSubsections. }
    • null. Defaults to null.

  • heading_md (string or null, optional)
    Markdown or HTML formatted string for the heading. The value can be any length, so can even be extended to create a block of MD/HTML content. Defaults to null.

TabSubsection

Description

A content subsection, to be used for dividing content into subsections, each with a title. e.g. "Short-read", "Long-read", "Hybrid".

Properties

  • id (string, required)
    Unique identifier for the subsection.

  • title (string, required)
    Title of the subsection.

  • content (array of TabItem, required)
    Content of the subsection.

TabItem

Description

A content item, which is typically a Galaxy tool, workflow or tutorial. In the UI, this will be rendered as an "accordion" item. If button/view links are not specified, no buttons will be displayed. For tools and workflows, inputs can be used to describe the essential inputs.

Properties

  • title_md (string, required)
    Markdown or HTML formatted title.

  • description_md (string, required)
    Markdown or HTML formatted description.

  • button_link (string or null, optional)
    URL for the button link. Defaults to null.

  • button_tip (string or null, optional)
    Tooltip text for the button. Defaults to null.

  • button_md (string or null, optional)
    Markdown or HTML formatted button label. Defaults to null.

  • button_icon (ButtonIcon or null, optional)
    Icon for the button. Defaults to null.

  • view_link (string or null, optional)
    URL for the view link. Defaults to null.

  • view_tip (string or null, optional)
    Tooltip text for the view. Defaults to null.

  • view_md (string or null, optional)
    Markdown or HTML formatted view label. Defaults to null.

  • view_icon (ButtonIcon or null, optional)
    Icon for the view. Defaults to null.

  • exclude_from (array of strings or null, optional)
    List of items to exclude. Defaults to an empty array.

ButtonIcon

Description

Available icons for buttons.

Enum Values

  • run
  • tutorial
  • social
  • help
  • view
